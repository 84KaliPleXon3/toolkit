  • This page shows and describes the different widgets of the JUNGER toolkit. See some widgets in action, test things in a terminal (Chrome: CTL+SHIFT+J, FF: Firebug + F12) or just get some information about the different options of the API.

    Please note #1: Options marked with an anchor sign ⚓ are only available on build time and remain static after the widget was created.

    Please note #2: This toolkit is in the origination process so it may contain some bugs. If you find any of them please send a mail to schmidt@boomshop.net. Thank you very much.

  • Button is a simple, clickable widget to trigger funcions. They fire a couple of click-related events and consist of a label and an icon. Buttons are used as a base to build different other widgets from.

    Use b to play with the example in a terminal: b.set("state", true) (toggles the state of the widget)

    OptionDescriptionDefault
    class ⚓ A CSS class to add on build time [empty]
    id ⚓ An ID to add on build time [empty]
    container A container element to inject the state into false
    label A text or HTML snipped used as a label for the button. Set to false to hide the label element. false
    icon A URL to an icon to show on the button. Set to false to hide the image element. false
    state Buttons can represent a state (like a toggle button) which is indicated by the class ".active" false
    state_color If the state is true, set a custom background color for the label. false
    Element (class)Description
    button.element ("div.toolkit-state") The base element of the widget. It draws the background color and acts as a container fo the other elements.
    button._label ("div.toolkit-label") The text label of the button
    button._icon ("img.toolkit-icon") An icon to display
    EventDescription
    clicked This event is fired when the button was clicked.
    mousedown This event is fired when the mouse button was pressed.
    mouseup This event is fired when the mouse button was released.

  • LevelMeter is a fully functional display of numerical values. They are enhanced MeterBases containing a clip LED, a peak pin with value label and hold markers. LevelMeters have some automatically triggered functionality like falling and resetting all kinds of values after a time. All additional elements can be set automatically as soon as the value rises above them.

    Use meters.[title] to play with the examples in a terminal: meters.mvr.set("value", 12) (sets the value of the vertical meter on the left to 12)

    Please don't expect too much performance with this poorly hacked demo on this overloaded page..

    top The value for the top hold, false = set to value. false
    bottom The value for the bottom hold, false = set to value. false
