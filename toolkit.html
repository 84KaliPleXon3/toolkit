  • toolkit. by Jünger is a graphical toolkit written in JavaScript for building audio related user interfaces running in modern browsers.

    This page shows and describes the different widgets of the Jünger toolkit. See some widgets in action, test things in a terminal (Chrome: CTL+SHIFT+J, FF: Firebug + F12) or just get some information about the different options of the API.

    Please note #1: Options marked with an anchor sign ⚓ are only available on build time and remain static after the widget was created.

    Please note #2: Options marked with an exchange arrow sign ⇄ point out a DOM element which was delegated to the main widget instance (see Widget.delegate() for more details).

    Please note #3: Options marked with an exchange harpoon sign ⇌ point out a DOM element which was classified to the main widget instance (see Widget.classified() for more details).

    Please note #4: This toolkit is in the origination process so it may contain some bugs. If you find any of them please send a mail to schmidt@boomshop.net. Thank you very much.

  • Widgets

    Widgets are fully functional elements to build user interfaces. They typically rely on other elements like Modules and Implements and are somehow based on Widget or other Extends.

  • Button is a simple, clickable widget to trigger funcions. They fire a couple of click-related events and consist of a label and an icon. Buttons are used as a base to build different other widgets from. Button extends Widget.

  • ResponseHandle is a controller inside a ResponseHandler, but can manually be used in FrequencyResponse or Graph, too. ResponseHandles have various faces and functions to provide different controls for every application an alterable Graph may be used in:

    • Circular handles: used for e.g. parametric EQ's, normally draggable in x and y direction. Able to show z axis via its size.
    • Lines: Vertical lines ca be used to e.g. modify a shelving filter while horizontal lines restricted in width may be used as handles for a graphical EQ with fixed bands. They are able to show the z axis via their size.
    • Blocks: Blocks roughly act like lines but draw themself as rectangles filling the space to their boundary. They can't show any z axis except as value in the label.
    ResponseHandle is based on Widget and implements Ranges.
