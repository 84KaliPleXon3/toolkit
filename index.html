• toolkit is a graphical toolkit written in JavaScript for building audio related user interfaces running in modern browsers. toolkit is supposed to work in Chrome/Chromium, Safari, Firefox > 4 and Internet Explorer >= 9.

    toolkit offers some widgets of all day use like Window or Button but concentrates especially on things like FrequencyResponse, Knob or Gauge.

    This page shows and describes the different widgets of the toolkit See some widgets in action, test things in a terminal (Chrome: CTL+SHIFT+J, FF: Firebug + F12) or just get some information about the different options of the API.

    • Please note #1: Options marked with an anchor sign ⚓ are only available on build time and remain static after the widget was created.
    • Please note #2: Options marked with an exchange arrow sign ⇄ point out a DOM element which was delegated to the main widget instance (see Widget.delegate() for more details).
    • Please note #3: Options marked with a sparkle sign ❇ point out a DOM element which was classified to the main widget instance (see Widget.classified() for more details).
    • Please note #4: Options marked with a pencil sign ✎ point out a DOM element which was stylized (see Widget.stylize() for more details).
    • Please note #5: This toolkit is in the origination process so it may contain some bugs. It is developed mainly using Chrome so there may be undetected flaws in other browsers. If you find any of them please send a mail to schmidt@boomshop.net. Thank you very much.

  • Widgets

    Widgets are fully functional elements to build user interfaces. They typically rely on other elements like Modules and Implements and are somehow based on Widget itself or other widgets.
